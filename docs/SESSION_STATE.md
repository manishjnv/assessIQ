## Decisions resolved this session

- **P3.D9 (SMTP provider) → Resend (default), via nodemailer-over-generic-SMTP.** User overrode the plan's AWS-SES default after Phase 0 review. Rationale captured in `docs/06-deployment.md` § SMTP provider: faster onboarding (10 min vs ~1hr SES IAM dance), free tier covers SOC pilot volume (3K/month), nodemailer SMTP transport keeps the swap mechanical (just change `SMTP_URL`). DPDP residency caveat: Resend lacks India region; for SOC pilot with non-PII content this is OK; production with candidate names/scores in email body needs legal sign-off before flipping. The `tenants.smtp_config.provider` enum stays open-ended (`'resend'|'ses'|'sendgrid'|'smtp'`) so per-tenant native-SDK overrides remain a Phase 4 swap without schema change.
- **P3.D12 (retry schedule) → literal `[1m, 5m, 30m, 2h, 12h]`** via custom BullMQ backoff strategy. NOT exponential. The user's task brief asked for "exponential backoff with jitter" but the plan correctly pins this as a published API contract (`docs/03-api-contract.md:324`) — host integrators have already coded their dedupe windows against it. Plan wins per global CLAUDE.md.
- **P3.D13, P3.D14, P3.D16** — accepted as-pinned in plan; no overrides.
